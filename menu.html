Modelo de Dados

Modelo Hierárquico


Foi o primeiro a ser considerado como modelo de dados.

Neste modelo os dados são extruturados em Hierárquias ou árvore.


Os nós das hierárquias contêm ocorrencias de registros, onde cada registro é uma colecção de campos (atributos), cada um contendo apenas uma (1) informação .

O registro da hierárquia que procede a outros é o registo-Pai, os outros são chamados R.Filhos.


Uma ligação é uma associação entre dois registros. O relacionamento entre um R.Pai e um R.Filho posssui cardinalidades 1:N;

Modelo Relacional


Foi criado por Edgar Fank Cood em 1970 para descrever como as bases de dados devem funcionar.O M.Relacional revela-se ser o mais flexivel e adequado para solucionar os vários problemas na implementação de BD.

A extrutura fundamental do MR é a tabela. Uma tabela é constituida por um (1) ou mais atributos (campos). Cada instância da tabela (linhas) é chamada de (dupla) "registro"

Modelo Orientado ao objecto

Modelo orientado a objecto as bases de dados orientados a objectos começaram a se tornar comercialmente viveis em meias de 1980.A razão para o seu surgimento foram os limites de armazenamento do modelo relacional.

Alguns exemplos de Modelo orientado ao objectos são: Os sistemas de informação Geográfica SIG com sistemas CAD e CAM que são q mas faceis construidos usando um tipo de dados complexos;

Hoje acredita-se que os bancos de dados orientados ao objecto serão usados em aplicativos especializados, em quanto os sistema relacionas continuam os negócios tradicionais onde as estruturas de dados baseado em relações são suficientes.
